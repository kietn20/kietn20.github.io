Dodge This!


Scratch Game

Description

The main idea of this game is to dodge anything that is flying towards your sprite.
The scoring system was a certain time(seconds) that the player must achieve to win the level.
I set the sprites flying towards the main sprite to fly in random directions and x's.
There are three types of difficulties: Easy, Medium, and Hard. When the flying sprites come
in contact with the main sprite, it will broadcast GAMEOVER, which will show a screen that says "GAMEOVER".


Reflection

I made this game because I realized the concept of this game include all the requirements of the criteria(Ex: random block, MORE blocks,
ranges in difficulties, etc). What inspired me to make this game was "World Hardest Game 1,2,3". Two successful points in the development of my game was how was I going use the MORE BLOCKS in my script and how to increase the difficulties without making it impossible to win. Two obstacles in the development process of my game would be the fault of a BROADCAST bug and how to RESET a level when you lose. I overcame the first problem by deleting the entire sprite and starting over ,and I used a varible to differ which level would be broadcast when the reset button is pressed. If I had
more time or resources, then I would improve on the movements/costumes of the sprites.


This algorithm contributes significantly to the game because it moves the main sprite to dodge all of the other sprites. When the level start, set the main sprite coordinates to (0,0) or the center of the screen. There is a forever script that forever states that if DOWN, UP, RIGHT, or LEFT are pressed then it would move in that direction. When it is moving, it changes costumes to make it seems like it's actually moving instead of just sliding.
